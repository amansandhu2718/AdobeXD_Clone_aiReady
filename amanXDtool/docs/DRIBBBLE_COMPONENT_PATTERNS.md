# Dribbble Component Patterns

Use this as compact local visual knowledge for modern UI generation. Never browse or open Dribbble while generating. Do not copy shots exactly. Use these patterns as inspiration for original amanXDtool designs.

## Common Dribbble-Style Screens

### Mobile App Home

Use for food, wellness, fintech, ecommerce, travel, smart home, education, dating, and social apps.

Patterns:

- Status-safe header with greeting, avatar, notification icon, or location.
- Large search or primary action near the top.
- Horizontal category chips.
- Featured card or promotional hero card.
- Scrollable product/content cards.
- Bottom tab bar with 3-5 items.
- One highlighted active state.

Design feel:

- Friendly spacing.
- Large rounded cards.
- Realistic sample data.
- Image-filled cards where category is visual.

### Mobile Dashboard

Use for finance, fitness, analytics, habit tracking, smart home, freelancer/productivity, crypto, and health.

Patterns:

- Top header with profile/action.
- Big balance/progress/summary card.
- Metric cards in 2-column grid.
- Mini chart cards.
- Transaction/activity list.
- Floating CTA or bottom nav.

Design feel:

- Strong numeric hierarchy.
- Clean charts with simple lines/bars/arcs.
- Muted surfaces with one accent color.
- Compact but not cramped.

### SaaS / Web Dashboard

Use for admin, analytics, CRM, finance, AI tools, project management, ecommerce admin.

Patterns:

- Sidebar with active nav item.
- Top bar with search, filters, date range, account.
- KPI cards row.
- Main chart/table area.
- Secondary insight/activity panel.
- Status chips and progress indicators.

Design feel:

- Dense Productive style.
- Quiet palette.
- Clear data hierarchy.
- Implementation-friendly grid.

### Landing Page

Use for mobile apps, SaaS, creator tools, marketplaces, AI products, agencies, and startups.

Patterns:

- Header with logo, nav, CTA.
- Hero headline with one strong promise.
- Product mockup, collage, image card, or dashboard preview.
- Trust metrics or logos.
- Feature cards or category cards.
- App-store/primary CTA area.

Design feel:

- First viewport should feel complete.
- Use one memorable visual composition.
- Make the next section visible or implied.

### Marketplace / Food Delivery

Use for food, ecommerce, local services, travel, real estate, jobs, events.

Patterns:

- Location/search-first hero.
- Category chips or icon tiles.
- Listing cards with image, title, rating, time/price/status.
- Offer badges and promo strips.
- Map/list hybrid for local discovery.

Design feel:

- Image-heavy.
- Warm, appetizing, or commerce-friendly palette.
- Cards need realistic metadata.

## Component Patterns Designers Commonly Use

### Bento Grid

- Mix large and small cards.
- One dominant feature card.
- Use varied card sizes but consistent gutters.
- Good for landing pages, AI tools, SaaS features, portfolios.

### Hero Mockup Stack

- Layer phone, dashboard, card, or image previews.
- Use shadows and overlap carefully.
- Good for product launches and app landing pages.

### Glass / Frosted Cards

- Use translucent-looking light surfaces only if readability remains strong.
- Pair with soft gradients or image backgrounds.
- Best for fintech, crypto, music, travel, AI, event/ticketing.
- Avoid overusing it in serious admin tools.

### Gradient Accent Surface

- Use gradient on one hero card, CTA strip, or featured metric.
- Keep text contrast high.
- Do not make the entire UI one gradient.

### Floating Stats

- Small metric pills around hero/mockup.
- Useful for SaaS, fitness, finance, AI, productivity.
- Keep them aligned and purposeful.

### Category Chips

- Use icon + label or label-only chips.
- Active chip needs distinct fill/border.
- Use realistic labels.

### Product / Listing Card

- Image top or side.
- Title, subtitle, metadata, rating/status.
- Optional price/CTA.
- Good cards are content-rich but still scannable.

### Empty / AI Prompt State

- Central input or command bar.
- Suggested prompts/actions.
- Small trust/status indicators.
- Useful for AI tools and app builders.

### Timeline / Activity Feed

- Avatar/icon, event title, metadata, status.
- Use subtle connector lines only when helpful.
- Good for dashboards, project apps, CRMs.

### Progress Ring / Mini Chart

- Use simple chart visuals built from shapes/lines.
- Pair with a number and label.
- Avoid pretending to show complex data if not needed.

## Visual Detail Checklist

Add at least 5 of these to avoid generic output:

- real logo/name treatment
- active nav state
- useful search/filter
- category chips
- image-filled card
- rating/status/price pill
- metric with realistic number
- mini chart/progress visual
- CTA with icon
- offer/notification badge
- grouped list row
- hero mockup/card stack
- empty/prompt suggestions

## Domain Pattern Hints

Food delivery:

- search by dish/restaurant
- location selector
- cuisine chips
- restaurant cards
- rating, time, offer, distance

Fintech:

- balance card
- card visual
- transaction list
- spending chart
- secure/verified chips

Healthcare/wellness:

- calm palette
- appointment or habit cards
- progress rings
- doctor/coach/avatar cards
- reminders

AI/SaaS:

- command input
- workflow cards
- usage/status metrics
- generated preview panel
- activity stream

Ecommerce:

- product image cards
- filter/sort chips
- price/rating
- cart CTA
- promo strip

Travel/real estate:

- image-heavy cards
- map/list hint
- price/location/rating metadata
- saved/favorite action

## Do Not Do

- Do not create blank wireframes unless requested.
- Do not use generic labels like `Card Title`.
- Do not create every card the same size.
- Do not ignore imagery for visual domains.
- Do not export before grouping and aligning components.
